## DB設計をするときに考えるステップ

[参考URL](https://qiita.com/kenzo-ta/items/4a3b0e0f0d4f37bb2dd8)

1. ユースケースを考える
ユースケース名：顧客が商品を注文する
概要　　　　　：ECサイトで商品を注文する
アクター　　　：顧客
事前条件　　　：ログインしている
トリガー　　　：カートに商品をいれ、カート画面で”注文手続き”ボタンを押す
基本フロー
注文画面で送付先、決済方法を入力する
注文内容の確認”ボタンを押す
注文内容を確認して、”注文確定”ボタンを押すと購入が完了する
備考　　　　　：”注文内容の確認”を押す際、入力をしていないと”注文内容の確認”ボタンを押せないようにする

2. ユースケースからエンティティの洗い出しをする。
